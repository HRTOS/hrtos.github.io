os_scheduling

所属模块：调度管理 | 类型：调度控制函数

函数简介

os_scheduling 是 HRTOS 中用于切换调度策略的控制函数， 支持在 MYOS 调度模式与 HRTOS 调度模式之间进行切换。

不同调度策略在任务调度方式、实时性以及系统响应能力上存在差异， 该函数允许开发者根据实际需求动态调整调度行为。 同时，函数内部包含时间基准修正机制，确保切换过程中系统时间保持连续。

函数原型

char os_scheduling(unsigned char id);

参数说明

参数 说明
id 0：使用 MYOS 调度
1：使用 HRTOS 调度

返回值

0 表示切换成功，-1 表示当前状态不允许切换（例如重复设置）。

核心机制解析

1. 调度模式控制

系统通过内部标志位控制调度方式，当选择 MYOS 模式时， 调度行为更加简单；当选择 HRTOS 模式时，启用完整调度机制。

2. 中断控制机制

调度策略的切换依赖于定时中断控制（如 OS_INTERRUPT_T0）， 通过启用或关闭相关中断来改变调度触发方式。

3. 时间片备份与恢复

在切换调度模式时，系统会保存当前时间片，并在恢复时重新同步：

OS_TIME_XY += (OS_TIME_ONCE_BF - OS_TIME_ONCE); OS_TIME_ONCE = OS_TIME_ONCE_BF;

该设计可以避免调度模式切换带来的时间漂移问题， 保证延时函数和定时任务的正确性。

调度策略对比

策略 特点 适用场景
MYOS 结构简单，开销低 低复杂度系统
HRTOS 支持优先级调度，实时性强 多任务实时系统

使用示例

void main() { // 切换到 HRTOS 调度模式 os_scheduling(1); while(1) { // 系统运行 } }

注意事项

调度模式切换应在系统初始化阶段完成，不建议在运行过程中频繁切换。
切换调度策略会影响任务执行顺序，应确保系统逻辑兼容当前模式。
