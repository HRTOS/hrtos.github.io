os_rtos

所属模块：调度管理 | 类型：系统控制函数

函数简介

os_rtos 是 HRTOS 中用于控制系统运行模式的核心函数， 可在“普通模式（非RTOS）”与“实时调度模式（RTOS）”之间进行切换。

在普通模式下，系统按照单线程或简单循环方式运行； 在 RTOS 模式下，调度器接管 CPU，通过任务切换实现多任务并发执行。 该函数同时内置时间补偿机制，用于保证模式切换过程中系统时间的连续性与准确性。

函数原型

char os_rtos(unsigned char id);

参数说明

参数 说明
id 0：关闭RTOS（进入普通模式）
1：开启RTOS（进入实时调度模式）

返回值

0 表示切换成功，-1 表示当前状态与目标状态不匹配（例如重复开启或关闭）。

核心机制解析

1. 状态判断机制

函数首先判断当前 RTOS 状态，仅在状态变化时才执行切换， 避免重复操作带来的系统不一致问题。

2. 时间片备份机制

在关闭 RTOS 时，系统会保存当前时间片（OS_TIME_ONCE）， 用于后续恢复，确保时间连续。

3. 时间补偿机制（核心）

在重新开启 RTOS 时，系统会通过以下方式修正时间偏移：

OS_TIME_XY += (OS_TIME_ONCE_BF - OS_TIME_ONCE); OS_TIME_ONCE = OS_TIME_ONCE_BF;

该机制可以有效避免模式切换导致的时间跳变问题， 保证定时器、延时任务等逻辑正常运行。

使用示例

void main() { // 启动RTOS调度 os_rtos(1); while(1) { // 系统运行 } }

典型使用场景

  • 系统初始化阶段开启 RTOS
  • 调试阶段临时关闭调度器
  • 低功耗模式下暂停任务调度

注意事项

不建议频繁切换 RTOS 模式，否则可能导致系统时间误差累积。
在关闭 RTOS 时，所有任务调度将暂停，需确保系统仍可正常运行。
