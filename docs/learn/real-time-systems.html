实时系统基础（Real-Time Systems）

实时系统是一种以“时间约束”为核心正确性标准的计算模型。 系统正确性不仅取决于结果是否正确，还取决于结果是否在截止时间内完成。

Time Constraint Deterministic System HRTOS Foundation

一、实时系统定义

实时系统要求任务在规定时间窗口内完成，否则结果失效或系统行为错误。

正确性 = 逻辑正确性 + 时间正确性 截止时间：执行时间 ≤ 截止时间 超时 = 系统级错误

实时系统的核心边界是“时间约束是否可验证”。

二、时间约束模型

在实时系统中，时间是核心资源，所有任务都必须映射为时间约束模型。

任务模型： 任务 =（开始时间，执行时间，截止时间） 约束： 开始时间 + 执行时间 ≤ 截止时间

实时系统设计本质是时间预算分配问题，而不是单纯任务实现问题。

三、实时系统分类

硬实时

超时即系统失败，必须保证严格时间上界。

软实时

允许延迟，性能随延迟下降。

固实时

超时结果失效，但系统继续运行。

非实时

以吞吐量与公平性为核心。

四、关键评价指标

1. WCET（最坏执行时间） 2. WCRT（最坏响应时间） 3. 中断延迟上界 4. 调度确定性 5. 抖动（Jitter）

实时系统设计必须基于最坏情况分析，而不是平均性能。

五、系统执行模型

中断触发 → 调度决策 → 任务选择 → 上下文切换 → 截止时间验证

系统正确性依赖整条执行链的时间上界，而不是单点性能。

六、HRTOS中的位置

实时系统是HRTOS的约束层，它定义所有执行机制必须满足的时间规则。 系统设计目标是减少时间不确定性扩散。

体系结构： 应用 ↓ 任务 ↓ 调度 ↓ 中断 ↓ 实时约束层 ↓ 时间确定性模型

七、总结模型

RTOS： 系统 =（任务，调度器，中断，约束） ∀ 任务：完成时间 ≤ 截止时间 ∀ 中断：延迟存在上界约束 ∀ 调度器：行为具有确定性

实时系统本质是“时间约束计算模型”，而不是软件类型分类。
