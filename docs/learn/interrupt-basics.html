中断基础（Interrupt Basics）

中断机制是实时操作系统（RTOS）中最底层、同时也是最具决定性的执行机制之一。 它不是单纯的“事件通知方式”，而是整个系统时间模型的入口点。 在RTOS中，所有实时行为的发生，本质上都可以追溯到中断触发与中断响应链路。

Interrupt System Real-Time Core Deterministic Response

一、中断的本质定义

中断（Interrupt）是一种打断CPU当前执行流的机制，使处理器从正在执行的任务中脱离， 转而执行一个被预定义的“紧急处理路径”（ISR，Interrupt Service Routine）。

但从系统角度看，中断的意义远不止“打断执行”。 它的本质是：将异步事件转换为可执行的同步控制路径，从而让系统具备响应外部世界的能力。

如果没有中断，CPU只能通过轮询方式检测事件，这会导致资源浪费与响应延迟不可控。 中断机制的出现，使“事件驱动系统”成为可能。

二、中断在RTOS中的角色

在RTOS中，中断不只是硬件机制，而是整个系统调度体系的触发源。 可以说，中断是“调度的上游输入系统”。

RTOS调度并不是主动运行的，而是被事件驱动的：

中断 → 唤醒任务 → 修改就绪队列 → 触发调度器 → 发生上下文切换

因此，中断的设计质量直接决定系统的实时性上限。 如果中断路径不可控，整个RTOS的确定性模型将失效。

三、中断执行完整流程

[ 中断执行模型 ] 1. 外部/内部事件发生（定时器 / 外设 / 软件） 2. 中断控制器检测到中断请求 3. CPU暂停当前任务执行 4. 保存关键上下文（PC / SP / 寄存器） 5. 跳转至中断向量表 6. 执行ISR（中断服务程序） 7. 清除中断标志位 8. 恢复上下文 9. 返回被打断任务或触发调度

需要注意的是，在RTOS中，第8步之后并不一定立即返回原任务， 可能会触发调度器，从而切换到更高优先级任务执行。

四、中断的核心组成结构

中断源（Interrupt Source）

产生中断信号的实体，包括外设、定时器、DMA或软件触发事件。

中断控制器（NVIC / PIC）

负责中断优先级管理、屏蔽控制与中断分发，是中断系统的调度核心。

中断向量表

将中断号映射到ISR入口地址，实现快速跳转执行。

ISR（服务程序）

实际处理中断事件的代码路径，要求执行时间极短且确定。

五、中断上下文与保存机制

当中断发生时，CPU必须保存当前执行状态，这个状态被称为“上下文（Context）”。 如果不保存上下文，任务将无法恢复到原始执行位置。

必须保存的关键状态： - 程序计数器（PC） - 栈指针（SP） - 通用寄存器 - 状态寄存器（PSR） - 可选：FPU / SIMD寄存器

在RTOS设计中，上下文保存必须满足两个条件： 完整性与可预测性。任何不可控的额外保存都会破坏实时性边界。

六、中断与调度的关系

中断并不会直接“运行任务”，但它可能改变任务执行顺序。 当中断唤醒更高优先级任务时，系统必须立即进行调度决策。

中断 → 修改任务状态 → 更新就绪队列 → 触发调度器 → 可能发生上下文切换

因此，中断路径与调度器之间存在紧密耦合关系。 在设计RTOS时，这条链路必须被严格优化，否则系统延迟不可控。

七、中断嵌套与优先级机制

现代RTOS支持中断嵌套，即高优先级中断可以打断低优先级中断执行。 这使系统具备更强的实时响应能力，但也引入复杂性。

优先级机制

中断按照优先级排序，高优先级可抢占低优先级。

嵌套执行

允许多个中断层级叠加执行，但必须控制最大深度。

八、中断对实时性的影响

中断延迟（Interrupt Latency）是RTOS最重要的性能指标之一。 它定义了从事件发生到ISR开始执行之间的时间差。

同时，中断嵌套过深会导致系统不可预测性增加， 从而破坏实时系统的“最坏情况时间上界”模型。

实时系统核心约束： 中断延迟 ≤ 上界限制 ISR 执行时间 ≤ 确定性上限

九、本质总结

中断的本质不是“响应事件”，而是“建立时间驱动系统的入口机制”。 它连接了外部世界与RTOS内部调度逻辑，是整个实时系统的起点。

中断 = 事件 → 系统状态变化 → 调度触发 核心价值： - 让系统具备异步能力 - 控制时间进入路径 - 构建实时性基础

系统关联路径（System Execution Chain）

中断并不是独立机制，而是RTOS执行链路的一部分。建议按以下路径理解系统行为：

→ 实时问题的本质（时间作为正确性约束） → 确定性约束（系统行为必须可预测） → 上下文切换机制（任务执行迁移） → 调度器行为模型（任务如何被选择） → 中断流程系统视图（系统级执行路径）

该路径代表RTOS的完整执行链：事件 → 中断 → 调度 → 切换 → 执行
