IPC模型（Inter-Process Communication Model）

定义任务间通信与同步结构，包括数据传递方式与调度协作关系。

系统架构 IPC 通信模型

概述

在 HRTOS 中，任务之间不能直接访问彼此状态，必须通过 IPC 机制完成通信或同步。

IPC 提供消息队列、信号量、事件组等机制，用于任务间数据交换与执行协调。

IPC 的核心不仅是数据传输，更是任务执行顺序与同步控制。

工作机制

消息传递：队列 / 邮箱（异步通信） 同步机制：信号量 / 互斥锁（执行控制） 资源控制：共享内存 + 同步保护

IPC 用于协调任务执行顺序，避免竞态与不确定行为发生。

关键结构

消息队列：任务间数据通道 信号量：资源互斥与同步控制 事件组：多事件组合触发 邮箱：定向消息通信 共享内存：高速共享（需同步保护）

不同机制对应不同粒度的通信与实时性需求。

运行模型

任务A生成数据 → 写入IPC对象 → 调度器切换任务 → 任务B被唤醒 → 读取数据并处理

IPC 触发任务唤醒，由调度器决定执行顺序与时机。

扩展机制

优先级继承：避免优先级反转 阻塞队列：资源不可用时挂起 超时机制：避免死锁等待 广播通知：多任务同步唤醒

扩展机制用于提升实时性与系统稳定性。
