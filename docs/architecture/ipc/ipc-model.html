IPC模型（Inter-Process Communication Model）

IPC模型定义系统中任务之间通信与同步的整体结构，包括数据传递方式、同步机制与系统级抽象关系。

系统架构 IPC 通信模型

概述

在HRTOS中，任务之间不能直接共享执行状态，必须通过IPC机制完成数据交换与同步控制。

工作原理

IPC模型将通信抽象为三类核心机制：消息传递、同步机制与共享资源控制。

IPC的核心目标不是“传输数据”，而是“协调执行顺序”。

关键结构

消息队列（Message Queue） 信号量（Semaphore） 事件组（Event Group） 邮箱（Mailbox） 共享内存（Shared Memory）

运行模型

发送任务 → IPC机制（队列/信号量）→ 内核调度 → 接收任务唤醒 → 数据消费 → 状态同步

扩展说明

高级IPC模型支持优先级继承、阻塞队列、超时机制与多任务广播机制。
