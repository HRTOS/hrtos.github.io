HRTOS 调度核心

调度模型（Scheduling Model）

调度模型定义系统如何从多个任务中选择执行对象，是实时操作系统中决定执行顺序与响应延迟的核心机制。

Scheduling System Model Time Deterministic Kernel Preemption Control System

调度模型定义

调度模型本质是一个系统状态选择函数： S(t) = f(中断，优先级，系统状态，资源可用性)

  • 输入可枚举（Requirement for analyzable state space）
  • 调度点有限（Bounded scheduling points）
  • 状态更新顺序一致（Deterministic state transition order）

它在每次事件触发点重新计算系统下一执行状态，包括任务释放、中断退出、超时与 IPC 唤醒。

调度流程模型

调度控制闭环

1. 中断 / 事件触发

↓

2. 就绪队列更新

↓

3. 优先级评估

↓

4. 调度决策

↓

5. 上下文切换

↓

6. 任务执行

  • 就绪更新：Event → Ready Set
  • 优先级评估：Ready Set → Execution Decision
  • 上下文切换：Decision → CPU State Transition

调度过程由就绪集、运行集、阻塞集与超时集构成。 核心是把外部事件快速转换为 CPU 所有权变化。

调度类型

抢占式调度

高优先级任务可打断低优先级任务。

时间片调度

同级任务轮转执行。

实时调度

保证截止时间可达。

事件驱动调度

中断/信号触发执行。

实际系统通常混合使用多种调度方式，不同负载区域对应不同策略。
