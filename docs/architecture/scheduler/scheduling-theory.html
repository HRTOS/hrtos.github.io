调度理论（Scheduling Theory）

RTOS 的核心不是调度机制，而是调度可证明性（Schedulability & Determinism）。

Temporal Determinism WCET Analysis Feasibility Model

1. 核心定义

调度理论研究在有限 CPU 资源下，任务集合是否满足时间约束。

本质是一个可行性问题：

∀Ti : FinishTime(Ti) ≤ Di

调度结果不是“优化最优”，而是“是否可证明成立”。

2. 形式化模型

Ti = (Ri, Ci, Di, Pi)

Ri：释放时间 / Ci：WCET / Di：截止时间 / Pi：优先级

CPU 利用率约束（简化）：

Σ(Ci / Ti_period) ≤ 1

超过该边界 → 系统不可调度（not schedulable）

3. 调度核心原则

Worst-case determinism first

  • 设计基于 WCET，而非平均性能
  • 目标是 deadline guarantee，而非吞吐量
  • 必须支持最坏情况分析

4. 可调度性分析流程

  • 任务建模
  • WCET 计算
  • 响应时间分析
  • Deadline 检查

任何一步不可建模 → 理论失效（无法保证实时性）

5. 不可调度的本质原因

  • WCET 不可界定
  • 中断延迟不可预测
  • 锁竞争不可建模
  • 动态内存干扰
  • 缓存行为不可控

不可分析 = 不属于 RTOS 理论体系

6. 系统意义

调度理论用于定义系统是否属于“实时系统”，而不是实现任务执行。

  • 定义时间正确性边界
  • 提供系统可证明性基础
  • 连接中断 → 调度 → 内核执行
