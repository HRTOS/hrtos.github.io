调度器（Scheduler）

调度器负责决定系统中哪个任务在何时执行，是实时系统时间控制的核心组件。

Time Control Core Execution Decision Layer Deterministic Scheduling

模块概述

调度器将任务集、时间约束与资源状态统一映射为执行决策，是系统确定性的关键来源。

其核心关注点是响应时间上界、阻塞传播与抢占一致性，而不是吞吐量或平均性能。

调度器同时也是系统各模块的时间约束中心：任务设计、IPC、中断与内存模型都需与其对齐。

调度系统结构

Interrupt System（事件触发层） ↓ Scheduling Theory（可证明性层） ↓ Scheduling Model（执行决策层） ↓ Priority Model（优先级控制层） ↓ Kernel Execution（执行控制层）

三层结构分别描述：可调度性边界、运行时决策逻辑与优先级抢占关系。

系统意义

调度器定义系统是否具备时间确定性，是 RTOS 的时间中枢。

它不仅决定任务执行顺序，也决定系统是否仍然满足实时约束。

一旦调度模型失效，系统将从确定性系统退化为概率性行为系统。
