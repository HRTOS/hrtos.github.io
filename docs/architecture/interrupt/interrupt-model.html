中断模型（Interrupt Model）

中断模型描述实时系统如何组织中断源、优先级、中断处理程序与任务恢复机制。

系统架构 中断模型 实时系统

概述

中断模型定义系统如何接收外部事件，并在不破坏实时性的前提下完成响应与恢复。

工作原理

中断源产生事件后，由中断控制器进行优先级仲裁，CPU根据中断向量跳转至对应 ISR。

中断模型的核心目标是确保异步事件能够被及时响应，同时维持系统整体确定性。

关键接口 / 结构

Interrupt Source Interrupt Controller ISR Handler Scheduler Trigger Task Resume

运行流程

外部事件产生 → 中断请求 → 优先级仲裁 → ISR执行 → 调度判断 → 恢复任务或切换任务

扩展说明

高级中断模型支持中断嵌套、优先级抢占、软中断与延迟处理机制。
