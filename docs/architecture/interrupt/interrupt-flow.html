中断流程（Interrupt Flow）

中断流程定义了外部事件如何打断当前执行路径，并将控制权转移到中断服务程序（ISR）中处理。

系统架构 中断机制 实时系统

概述

中断是实时操作系统响应异步事件的核心机制，常见来源包括定时器、串口、GPIO、DMA 与硬件异常。

工作原理

当中断发生时，CPU 暂停当前任务，保存上下文，并跳转至对应 ISR。ISR 执行完成后，系统恢复原任务或触发调度决策。

中断延迟直接影响实时性保证，因此 ISR 应尽量短小、确定且可预测。

关键接口 / 结构

os_interrupt_enable() os_interrupt_disable() os_isr_register() os_isr_exit()

运行流程

中断触发 → 保存CPU上下文 → 执行ISR → 可选嵌套中断处理 → ISR退出 → 调度器检查 → 恢复任务或切换任务

扩展说明

高级中断系统通常支持优先级嵌套、中断屏蔽、延迟处理与中断向量管理。
