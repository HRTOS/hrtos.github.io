内存模型（Memory Model）

内存模型定义任务、内核与资源之间的运行时内存访问结构。 用于约束运行时访问行为。

Memory Model Runtime Constraint Kernel Interface

系统内存模型定义

HRTOS 内存模型描述系统在运行时对任务、内核与资源的统一访问规则。 它不是静态布局定义，而是运行时行为约束集合。

内存模型的核心作用是将“对象存在”转化为“对象如何被访问”， 并对访问路径施加确定性约束。

在实时系统中，内存访问本身即参与调度决策，因此必须纳入系统级设计。

内存访问语义

每一类内存区域都对应不同的访问语义与约束等级：

  • Task Space：仅允许任务私有访问，由调度器切换控制
  • Kernel Space：系统唯一控制区，仅通过内核入口访问
  • Shared Memory：通过 IPC 机制进行受控访问
  • Heap Region：动态分配区，访问时间不可变但必须受限

这些语义共同构成系统“可执行边界”，用于约束非法路径传播。

运行时约束模型

内存模型同时定义运行时约束条件，包括访问权限、生命周期与时序影响。

所有对象必须满足以下条件才能进入实时路径：

  • 访问路径可追踪
  • 生命周期可预测
  • 分配与释放成本有界
  • 跨任务访问必须显式同步

该模型用于确保系统在高负载情况下仍保持确定性行为。
