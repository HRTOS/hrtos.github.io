资源竞争（Resource Contention）

系统中“不可预测性”的结构性来源，而不是运行时现象

1. 核心定义（从现象到模型）

资源竞争不是“多个任务在抢资源”，而是：

多个执行流在时间维度上对同一系统状态产生不可交换影响

在 HRTOS 中，它的本质是： 时间路径分叉（Temporal Path Divergence）

Resource contention is not competition for resources, but divergence of execution paths in time space.

2. 因果链模型（Causal Chain Model）

资源竞争的完整因果链：

资源请求 → 等待队列形成 → 执行延迟 → 调度重排 → 时间偏移 → 系统抖动

关键点： 竞争不是问题，延迟传播才是问题

3. 三类资源的“时间属性”

① CPU（时间资源）

本质是“执行时间分配权”，竞争直接改变时间线。

② Memory（空间资源）

竞争会转化为访问延迟（Cache miss / Bus stall）。

③ Synchronization（状态资源）

Mutex / Semaphore 实际上是“时间阻断器”。

4. 时间确定性破坏机制

资源竞争通过三种方式破坏确定性：

✔ Execution delay injection（执行延迟注入）
✔ Scheduling order perturbation（调度顺序扰动）
✔ Path amplification（路径放大效应）

结果不是“慢”，而是： 时间边界失效

5. 阻塞结构不是链，而是图

传统理解：Blocking Chain

HRTOS 视角：Blocking Graph（阻塞图）

多个任务形成依赖图：

T1 → R1 ← T2 → R2 ← T3

问题不在链长，而在： 循环依赖 + 不可预测解锁点

6. 优先级反转 = 时间层级崩塌

优先级反转本质不是调度错误，而是：

时间优先级语义被资源锁重新排序

结果：

✔ 高优先级任务失去时间优先权 ✔ 调度模型与执行模型解耦 ✔ 系统进入不可证明状态

7. 内存竞争的隐藏层（Cache / Bus / DMA）

真正的竞争不在 heap，而在微架构层：

• Cache line bouncing • Memory bus arbitration • DMA burst preemption

这些因素导致： 同一函数执行时间变成随机变量

8. 可控系统 vs 不可控系统

HRTOS 的目标不是“消除竞争”，而是：

✔ 将竞争转化为可建模延迟 ✔ 将不可预测性转化为上界函数 ✔ 将系统行为变为可证明集合

9. 失控的最终形态

资源竞争失控不会“崩溃”，而是退化为：

• WCET 不存在 • Deadline 无法证明 • Scheduling becomes probabilistic • RTOS → GPOS 行为漂移

本质结论： 系统仍在运行，但已经失去系统性

语义总结

资源竞争不是一个“问题点”，而是： 时间确定性结构的破坏机制

Resource contention is not a problem to solve, but a structural mechanism that breaks temporal determinism.
