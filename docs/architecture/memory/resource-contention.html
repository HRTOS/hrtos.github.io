资源竞争（Resource Contention）

资源竞争描述多个执行流在同一时间窗口对系统资源产生的时间冲突与延迟传播现象， 是影响实时确定性的关键因素。

Timing Model Resource Conflict Determinism Risk

核心定义

资源竞争的本质不是“抢占资源”，而是执行路径在时间维度上的分叉与延迟传播。

当多个任务在同一资源上产生依赖时，系统行为从确定性执行转变为时间耦合系统。

资源类型与时间属性

CPU 竞争

表现为执行时间片分配变化，直接改变任务时间线。

内存竞争

表现为 Cache / Bus / DMA 延迟，影响访问时间稳定性。

同步资源

Mutex / Semaphore 引入阻塞，形成时间阻断结构。

阻塞传播

等待链会扩展为依赖图，引发全局抖动。

时间确定性破坏机制

资源竞争通过延迟注入、调度扰动与路径放大三种方式破坏系统确定性。

延迟不会局部结束，而会沿任务链传播并放大。

系统影响

资源竞争的最终影响不是性能下降，而是系统行为从可证明变为不可证明。

当 Worst Case Execution Time（最坏情况执行时间） 无法界定时，RTOS 将退化为不可预测执行模型。
