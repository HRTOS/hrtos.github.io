内核架构（Kernel）

内核是 HRTOS 的控制边界定义层，用于约束调度行为与执行合法性。

Execution Control Layer Scheduling Domain Interrupt Orchestration Deterministic Boundary

系统级定义（System Definition）

Kernel 的设计目标是定义系统执行的控制边界，而不是提供功能实现。

它决定：

  • 哪些任务可以运行
  • 何时允许调度发生
  • 哪些状态迁移是合法的

所有执行行为必须经过 Kernel 的统一判定。

Kernel 控制模型（Control Model）

Kernel 控制模型是一个事件驱动的闭环系统。

其基本结构为：

事件 → 决策 → 调度 → 状态转换

Kernel 在每个事件点重新评估系统执行合法性。

  • 事件触发系统状态变化
  • 内核进行调度判定
  • 系统进入新的执行状态
