HRTOS Architecture Model

系统边界（System Boundary Layer）

定义 Kernel 与外部环境之间的可计算分割结构，用于约束实时系统的可执行范围。

1. 边界模型定义

系统边界定义 Kernel 与外部世界之间的可计算约束空间。

B = {Tb, Rb, Ib}

Tb = 时间约束
Rb = 资源约束
Ib = 不确定性约束

2. Kernel 在边界中的角色

Kernel = 边界执行单元

负责保证所有执行行为符合系统边界约束。

禁止：无界阻塞 / 不确定动态分配 / 越权访问

3. 系统域结构

调度域

已注册任务 + ready/running 状态 + 优先级约束集合

时间域

任务执行必须满足确定性时间约束（Tb）

资源域

内存、CPU、外设资源的可控分配边界（Rb）

不确定性域

对外部事件扰动的隔离与约束（Ib）

4. 隔离机制

时间隔离 / 内存隔离 / 中断门控 / 优先级封顶

5. 系统意义

没有系统边界，就没有可预测的 RTOS 行为。

6. 语义总结

系统边界 = Kernel 可执行约束的上界
