内存分配（Memory Allocation）

内存分配机制用于管理系统运行过程中的内存申请与释放， 支持静态分配与动态分配，在实时系统中提供确定性与可控性内存使用模型。

Memory RTOS Core Resource Management

概述

内存分配（Memory Allocation）是RTOS内核中的基础资源管理机制， 用于为任务、队列、缓冲区等运行对象提供内存空间。

工作原理

HRTOS通常采用固定块分配或内存池机制，避免传统动态分配带来的不可预测延迟。 系统根据请求大小从预分配内存池中分配对应块。

在实时系统中，内存分配必须保证“时间确定性”，避免碎片化与不可控延迟。

关键接口 / 结构

os_mem_init() os_malloc() os_free() struct mem_pool { void* start; uint32_t block_size; uint32_t total_blocks; uint32_t free_blocks; };

运行流程

任务申请内存时，系统从内存池中查找可用块并分配； 释放时将内存块重新回收到空闲链表中。

扩展说明

HRTOS建议在关键实时路径中避免频繁动态分配， 优先使用静态分配或预分配内存池以保证实时性。
