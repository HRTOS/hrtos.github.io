内存管理（Memory Management）

内存管理模块统一管理RTOS运行期间的所有内存资源， 构建任务运行、通信与调度所需的基础内存体系。

Allocation Fragmentation Stack Isolation

模块定位

内存管理负责系统运行期间的内存分配、回收与使用约束， 为任务调度、通信机制与中断执行提供稳定的资源基础。

系统特性

HRTOS内存系统采用静态优先设计，避免不可控动态行为， 强调确定性、可预测性与低开销运行模型。

确定性分配

避免随机分配带来的不可预测延迟。

任务级隔离

通过软件边界限制任务访问范围。

低碎片设计

减少长期运行后的内存碎片累积。

栈安全检测

运行时检测SP边界异常并触发保护机制。

学习路径

建议学习顺序：内存分配 → 碎片控制 → 栈管理 → 任务隔离。
