时间片轮转调度（Round Robin）

时间片轮转调度是一种基于时间片的公平调度策略，通过固定时间片轮流分配CPU， 在保证系统响应性的同时提升任务公平性。

Scheduler Fair Scheduling Time Slice

概述

Round Robin（RR）调度为每个任务分配固定时间片， 当时间片用尽后自动切换到下一个就绪任务。

工作原理

系统维护一个循环就绪队列，任务按顺序执行。 当前任务时间片耗尽或主动让出CPU时，调度器切换到队列下一个任务。

Round Robin不依赖优先级，是典型的“时间驱动公平调度”模型。

关键接口 / 结构

os_rr_init() os_rr_set_timeslice() os_rr_scheduler_tick() struct rr_task { uint32_t timeslice; uint32_t remaining; };

相关接口文档： 优先级调度 / 实时调度 / 任务上下文切换 / 调度流程

运行流程

任务进入就绪队列 → 分配时间片 → 执行直到时间片耗尽 → 移至队列末尾 → 调度下一个任务。

扩展说明

在HRTOS中，Round Robin通常与优先级调度结合使用， 用于同优先级任务之间的公平调度。
