互斥锁（Mutex）

互斥锁用于保护临界资源的独占访问，防止多任务并发冲突，并支持优先级继承机制以避免优先级反转问题。

IPC通信 资源互斥 优先级继承

概述

互斥锁（Mutex）是一种用于保护共享资源的同步机制， 确保同一时刻只有一个任务可以访问临界区资源。

工作原理

当任务申请互斥锁时，如果锁未被占用，则获得访问权限； 如果已被占用，则任务进入阻塞状态并加入等待队列。

HRTOS中的Mutex支持优先级继承机制，用于解决高优先级任务被低优先级任务阻塞的问题。

关键接口 / 结构

os_mutex_init() os_mutex_lock() os_mutex_unlock() struct mutex { task_t *owner; task_list_t wait_list; int priority_inherit; };

运行流程

任务尝试获取Mutex → 若空闲则获得所有权 → 若被占用则阻塞等待 → 释放后唤醒等待任务并重新调度。

扩展说明

Mutex适用于保护共享资源，如外设驱动、全局数据结构、文件系统访问等关键临界区。
