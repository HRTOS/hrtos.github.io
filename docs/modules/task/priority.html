任务优先级（Task Priority）

任务优先级用于决定CPU调度顺序，是抢占式调度的核心依据。

概述

HRTOS优先级定义来自OS_TCB.cur_prio / base_prio， 由调度器在READY队列中进行比较选择。

优先级数值越大表示优先级越高（系统约定）， 高优先级任务可抢占低优先级任务。

系统优先级范围由 OS_SCHED_LEVEL_MAX 定义（0~10）， 高位用于内核/中断级任务，低位用于普通应用任务。

调度机制

调度器维护按优先级划分的READY队列， 始终选择最高优先级任务运行。

READY队列按优先级组织，调度复杂度为O(1)（位图选择）。

当更高优先级任务进入READY状态（os_task_unblock / 中断唤醒）， 当前任务会被立即抢占。

同优先级任务按时间片轮转（OS_TIME_ONCE_DEFAULT）。

调度规则

1. 选择最高cur_prio任务执行 2. 同优先级任务时间片轮转 3. 高优先级READY立即抢占 4. WAIT/SUSPEND不参与调度

WAIT态任务由资源事件唤醒进入READY。

关键接口

os_task_set_priority(id, prio) os_task_get_priority(id) OS_TCB: u8 base_prio; u8 cur_prio; u8 state;

set_priority修改cur_prio； get_priority返回当前调度优先级（可能为继承值）。

priority可能来源：base_prio 或 mutex继承提升值

运行流程

create task ↓ assign base_prio ↓ READY queue insert ↓ scheduler select highest prio ↓ RUNNING ↓ preempt / block / yield

调度触发点：任务就绪 / 中断退出 / 资源释放 / tick中断。

优先级继承

当高优先级任务等待mutex时， 持锁任务临时提升至更高优先级。

继承优先级存储在cur_prio中，释放锁后恢复base_prio
WAIT_MUTEX → inherit prio unlock → restore base_prio
