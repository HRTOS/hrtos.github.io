真实系统运行（Real System Run）

本实验展示 HRTOS 在实际运行中的完整链路： 任务创建 → 调度触发 → 中断响应 → 任务切换 → 任务执行

运行链路（真实内核行为）

os_task_create → Ready Queue → os_schedule_request → interrupt → os_enter_critical → scheduler → os_exit_critical

核心API映射（真实版本）

任务创建

os_task_create

调度请求触发

os_schedule_request

进入中断临界区

os_enter_critical

退出临界区

os_exit_critical

任务切换请求

os_task_yield

系统运行流程

任务创建阶段

os_task_create 将任务加入 Ready Queue。

调度触发

通过 os_schedule_request 或 Tick 中断触发调度。

中断进入

系统进入 ISR，执行 os_enter_critical 保护调度过程。

调度执行

调度器选择最高优先级任务。

任务切换

通过 yield 或系统调度完成任务切换。

系统意义

真实系统运行体现 RTOS 内核的完整协作机制，是理解调度器行为的基础模型。
