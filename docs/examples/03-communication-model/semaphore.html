信号量机制（Semaphore）

信号量用于任务同步与资源管理，通过计数机制控制并发访问， 实现阻塞、唤醒与调度协作。

机制定义

Semaphore = 资源计数 + 阻塞队列 + 唤醒机制 + 调度联动

在 HRTOS 中，信号量是最基础的同步工具，用于解决任务间资源竞争问题。

核心行为

申请资源

计数>0则获取成功，否则阻塞。

任务阻塞

无资源时进入等待队列。

释放资源

计数增加并唤醒等待任务。

任务恢复

被唤醒任务重新进入调度。

执行流程

1. 初始化信号量

设置初始资源数量。

2. 任务申请

尝试获取资源。

3. 阻塞等待

资源不足进入等待队列。

4. 释放资源

资源释放触发唤醒。

系统关系模型

Task → Semaphore → Scheduler → Task

演示代码


#include <HRTOS_KERNEL.H>
#include <HRTOS.H>

void task_A(void)
{
    while(1)
    {
        os_wait1(2);
    }
}

void task_B(void)
{
    while(1)
    {
        os_wait1(2);
    }
}

void main(void)
{
    os_scheduling(1);

    os_task((unsigned int)task_A, 0, 3, 0);
    os_task((unsigned int)task_B, 1, 2, 0);

    os_task_release(0);

    while(1)
    {
        os_nop();
    }
}

系统意义

信号量是 RTOS 并发控制的基础机制之一， 用于保证任务间同步执行与资源安全访问， 对系统稳定性与实时性具有关键作用。
