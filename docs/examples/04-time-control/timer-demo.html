定时器机制（Timer System）

Timer 基于 HRTOS Tick 系统实现周期触发与事件驱动回调， 用于实现延时任务、周期任务与事件触发机制。

机制模型

Timer = Tick Counter + Compare Match + Callback Dispatch
所有 Timer 行为依赖 os_tick_config / os_tick_get / scheduler dispatch

执行流程

1. Timer 注册到内核
2. Tick递增计数
3. Compare Match触发
4. 调度器执行回调任务

HRTOS Timer 示例代码


#include "hrtos.h"

#define TIMER_ID 0
#define EVENT_ID  1

/* Timer回调（建议：轻量 + 事件触发） */
void timer_callback(void)
{
    os_event_write(EVENT_ID);   // 推荐：通过事件解耦
}

/* 初始化 */
void system_init(void)
{
    os_event_init(EVENT_ID);

    /* 1000ms周期Timer */
    os_timer_create(TIMER_ID, os_ms_to_tick(1000), timer_callback, 1);
    os_timer_start(TIMER_ID);
}

/* 消费任务 */
void task_consumer(void)
{
    while(1)
    {
        os_event_wait(EVENT_ID, 0);  // 永久等待

        LED_EVENT = ~LED_EVENT;
    }
}

/* system entry */
void hrtos_main(void)
{
    system_init();

    os_task_create((unsigned int)task_consumer, 1, 2, 5);

    while(1)
    {
        os_nop();
    }
}

系统设计原则

Timer不做重计算

Timer回调应尽量短，仅用于触发事件。

推荐事件解耦

Timer → Event → Task，避免ISR阻塞。

Tick统一驱动

Timer全部依赖系统Tick，而不是独立计时。
