上下文切换流程（Context Switch Flow）

上下文切换是RTOS实现多任务并发的核心机制，用于在任务切换时保存当前执行状态并恢复目标任务状态。

System Kernel Mechanism Context Switch Real-Time

概述

上下文切换是CPU在不同任务之间切换执行时， 保存当前任务状态并恢复目标任务状态的过程。

工作机制

当系统发生调度事件时，内核触发上下文切换流程， 完成任务状态的保存与恢复。

上下文切换开销直接影响系统实时性，是RTOS核心性能指标之一。

触发条件

1. 时间片结束 2. 任务主动让出CPU 3. 任务进入阻塞状态 4. 高优先级任务就绪 5. 中断触发调度

执行流程

触发调度 ↓ 进入调度器 ↓ 保存任务状态 ↓ 选择最高优先级任务 ↓ 恢复目标任务状态 ↓ 继续执行

本质是“状态保存 + 状态恢复”的调度过程。

状态保存与恢复

系统在切换过程中需要保存当前任务运行状态， 并在恢复时重新加载目标任务状态。

通用寄存器 程序计数器 PC 栈指针 SP 状态寄存器
保存内容越多，上下文切换开销越高。

性能影响因素

1. 保存状态数量 2. 调度复杂度 3. 中断嵌套 4. 栈操作开销 5. CPU执行效率

HRTOS通过精简调度路径与状态保存策略降低切换延迟。

机制说明

支持基于中断与调度器触发的两种上下文切换路径， 以满足实时系统响应需求。
