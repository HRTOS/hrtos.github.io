# HRTOS 开发者文档

## 概述

本文档为维护和扩展 HRTOS（硬实时操作系统）内核的开发者提供全面的技术细节。HRTOS 专为 8051 系列微控制器设计，专注于低内存占用和实时性能。

## 文档结构

本文档涵盖 HRTOS 的以下方面：

- **内核概述**：系统架构和设计理念
- **目录分析**：项目结构和组织
- **调度器设计**：任务调度算法和机制
- **任务管理**：任务生命周期和控制
- **事件设计**：事件同步机制
- **队列设计**：消息队列实现
- **互斥锁设计**：具有优先级继承的互斥
- **信号量设计**：计数信号量实现
- **定时器系统**：基于时钟的定时服务
- **中断设计**：中断处理和 ISR 集成
- **内存布局**：8051 架构的内存组织
- **移植层**：硬件抽象接口
- **内核启动**：系统初始化序列
- **状态机**：任务和资源状态转换
- **模块依赖**：模块间关系
- **调用流程**：执行流程和调用图
- **数据结构**：核心内核数据结构
- **设计原则**：架构指南和约束
- **扩展指南**：添加新功能和模块
- **调试**：调试技术和工具

## 系统特性

- **目标平台**：8051 微控制器
- **内存模型**：DATA、IDATA、XDATA 段
- **任务类型**：标准任务（0-15）、快速任务（2）
- **优先级级别**：0-10（0-7 标准、8 快速、9-10 中断）
- **调度**：基于优先级的抢占式调度，可选时间片
- **IPC**：事件、消息队列、互斥锁、信号量、邮箱
- **内存管理**：静态分配，位图跟踪

## 快速参考

### 关键数据结构

- `OS_TCB`：任务控制块
- `OS_RESOURCE`：统一 IPC 资源对象
- `os_msgq_t`：消息队列结构

### 关键配置

- `OS_PROCESS_MAX`：最大标准任务数（16）
- `OS_FAST_TASK_MAX`：最大快速任务数（2）
- `OS_RESOURCE_MAX`：最大 IPC 资源数（8）
- `OS_MSGQ_MAX`：最大消息队列数（4）

### 内存布局

- 系统栈：0x35-0x39（5 字节）
- 中断栈：0x3A-0x41（8 字节）
- 用户栈：0x42-0x7F（62 字节）
- XDATA 全局池：0x0200-0x03FF（512 字节）
