内存管理（Memory Management）

所属模块：内核机制 | 类型：资源管理

概述

内存管理是实时操作系统中用于组织与分配运行资源的重要机制， 负责协调任务执行过程中对数据空间的使用与访问。

在 HRTOS 中，内存管理采用轻量级设计， 主要通过静态分配与结构化数组实现资源管理， 以降低运行开销并提高系统稳定性。

设计思路

HRTOS 并未采用复杂的动态内存分配算法， 而是通过预先分配的数据结构来管理系统资源， 从而避免碎片化问题并提升实时性。

  • ✔ 静态分配为主，减少运行时开销
  • ✔ 使用数组管理任务相关数据
  • ✔ 避免频繁申请/释放内存带来的不确定性

核心数据结构

系统通过多个全局数组维护任务状态与运行数据， 这些结构共同构成内存管理的基础。

OS_JINCHENG_SP[] // 各任务栈指针存储 OS_WAIT_DI[] // 任务延时计数 OS_PEOCESS_OK[] // 任务就绪状态标志

每个数组对应不同类型的系统资源， 在任务调度与执行过程中被统一管理。

工作机制

内存管理在系统中的典型工作流程如下：

  1. 系统初始化时分配各类全局资源
  2. 任务创建时分配对应的数据存储区域
  3. 任务运行过程中访问自身数据
  4. 调度器根据状态更新资源内容

与任务调度的关系

内存管理与任务调度密切相关， 调度器通过访问任务状态数组判断任务是否可运行， 并结合延时计数与资源状态实现调度决策。

设计特点

  • ✔ 结构简单，易于实现
  • ✔ 无动态分配，实时性更强
  • ✔ 适合资源受限的嵌入式系统
  • ✔ 与调度系统紧密耦合

注意事项

静态内存分配虽然稳定，但灵活性较低，设计时需合理规划资源规模，避免空间浪费或不足。
