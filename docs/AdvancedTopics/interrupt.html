中断系统（Interrupt System）

所属模块：内核机制 | 类型：实时响应

概述

中断系统是实时操作系统实现快速响应的核心机制， 用于在外部事件或定时触发时打断当前任务执行， 并转入特定处理流程。

在 HRTOS 中，中断不仅用于处理外设事件， 同时也是任务调度的重要触发方式， 尤其是基于定时器的时间片调度。

中断的作用

  • ✔ 响应外设事件（如IO、通信）
  • ✔ 驱动任务调度（定时器中断）
  • ✔ 实现时间片切换
  • ✔ 保证系统实时性

HRTOS 中断处理机制

HRTOS 通过统一的中断入口函数对不同中断进行分发， 并在定时器中断中执行任务切换逻辑。

i = os_inter(); // 获取中断号 if(i == 1) // 定时器中断（调度核心） { TF0 = 0; TH0 = 0; TL0 = 0; i = os_task_id(); os_task_protect(i); i = os_select(i); os_task_release(i); } else { if(i) i--; os_operate(my_inter[i]); }

中断触发与调度流程

  1. 定时器或外设触发中断
  2. 进入统一中断入口
  3. 判断中断类型
  4. 若为定时器 → 执行任务切换
  5. 若为外设 → 调用对应处理函数

与任务调度的关系

中断系统与任务调度紧密耦合， 调度器通常由定时器中断驱动， 从而实现周期性任务切换。

这种设计保证了系统能够在固定时间片内切换任务， 同时兼顾实时性与执行效率。

设计特点

  • ✔ 中断统一入口，结构清晰
  • ✔ 调度逻辑集中在定时器中断
  • ✔ 支持多中断源扩展
  • ✔ 响应速度快，适合实时系统

注意事项

中断处理应尽量简短，避免在中断中执行耗时操作，以免影响系统实时性与调度效率。
