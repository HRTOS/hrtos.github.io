3. 中断流模型（Interrupt Flow）

外部事件如何打断执行流并重构系统调度路径

核心定义

中断流描述外部事件如何打断当前执行流，并触发系统重新进入调度状态。

在 HRTOS 中，中断不是异常机制，而是时间模型的一部分。

Interrupt flow is part of the timing model, not an exception mechanism.

1. 中断触发

来源包括 Timer、IO、通信接口与异常事件。

必须满足：可预测 + 有界频率，否则无法进行实时分析。

2. 中断响应

CPU进入ISR并保存上下文。

响应延迟由：硬件延迟 + 屏蔽窗口 + 上下文保存构成。

RTOS核心目标是界定该延迟上界。

3. 中断嵌套

高优先级中断可抢占低优先级ISR，形成嵌套链。

嵌套深度必须有界，否则系统失去可分析性。

Nesting Depth ≤ Bounded Constant

4. 与调度的关系

中断可以直接触发调度器重新选择任务。

在抢占式RTOS中，中断是调度事件源。

5. 执行恢复

ISR结束后可能返回原任务或触发重新调度。

高优先级事件通常直接进入调度路径。

语义总结

中断流的本质是：时间执行被切割后重新组合为新的执行序列。

它决定系统是否能维持时间连续性与确定性。

Interrupt flow reconstructs execution time under external events while preserving determinism.
