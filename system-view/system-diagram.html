System Diagram

HRTOS 系统运行结构：执行流 × 时间 × 控制路径
HRTOS 系统不是线性流程，而是一个事件驱动 + 调度决策 + 时间约束共同构成的闭环系统。
下面的结构表示“控制关系”，不是“阅读顺序”。
[ Execution Foundation ]
01 Execution Flow
系统执行的基础路径结构：任务如何从触发进入 CPU 执行空间
本质：时间路径模型（Time Path Model）
↑ 被调度器驱动 / 被中断打断 ↓
04 Scheduler Cycle
系统控制中心：持续决定“谁可以执行”
本质：时间决策引擎（Time Decision Engine）
↑ 依赖状态系统 ↓
02 Task Lifecycle
系统中所有执行单元的状态集合
Ready / Running / Blocked 构成调度输入空间
↑ 被外部事件修改 ↓
03 Interrupt Flow
系统外部世界进入内核的唯一通道
本质：执行流重写机制
↑ 影响时间行为 ↓
05 Memory View
所有执行行为的隐性约束层
Cache / 分配 / 访问延迟 → 直接影响确定性
