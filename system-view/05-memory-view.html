5. 内存视图（Memory View）

内存如何成为实时系统的时间约束来源

核心定义

内存视图描述系统如何管理内存资源，并保证其不会破坏时间确定性。

在 HRTOS 中，内存不是静态资源，而是影响执行时间的时间变量源。

Memory is a timing-related system variable, not just a storage resource.

1. 内存分配模型

动态内存分配（malloc/free）在RTOS中是不可控风险源。

常见替代方案：

• 静态分配
• 内存池
• 固定块分配

核心原则：分配时间必须有界（Bounded Allocation Time）

2. 内存碎片问题

动态分配会产生碎片，导致分配时间逐渐不可预测。

这会引入时间漂移问题（Timing Drift）。

在实时系统中，这比空间浪费更严重。

3. Cache与访问不确定性

Cache命中/未命中会导致访问时间出现数量级差异。

内存访问因此变为随机变量，而非固定延迟操作。

需要通过缓存策略或最坏情况分析进行约束。

4. 内存隔离

多任务系统必须保证任务之间内存边界清晰。

否则会引入：

• 数据竞争
• 时间干扰

隔离不仅是安全问题，也是实时性问题。

5. 内存与时间耦合

内存行为的不确定性最终会转化为执行时间不确定性。

关系链：

Memory Behavior → Execution Time → System Determinism

语义总结

在实时系统中，内存不是空间资源，而是时间约束来源。

它是影响系统确定性的隐性关键变量。

Memory is a hidden timing constraint in real-time systems.
