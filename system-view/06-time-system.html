06 时间系统（Time System）

时间是实时系统的第一约束变量，而不是辅助机制

核心定义

时间系统不是计时模块，而是实时系统的约束框架。 所有调度、中断与任务执行，都必须满足“时间可预测性”。

在 HRTOS 中，时间不只是背景变量，而是系统正确性的直接判断依据。

Time system defines temporal correctness, not just timekeeping.

1. 三层时间模型

① 物理时间（Real Time）

外部世界的真实时间流，如传感器输入与物理变化，系统只能响应。

② 系统时间（System Time）

由 tick 或定时器构成的内部时间基准，是调度统一参考系。

③ 执行时间（CPU Time）

任务实际占用 CPU 的时间，是 WCET 分析核心。

2. Tick 与时间粒度

Tick 是时间离散化单位，决定系统调度精度。

• Tick 小 → 精度高但开销大
• Tick 大 → 开销低但延迟粗

本质是“时间分辨率 vs 系统成本”的权衡。

Tick defines scheduling granularity and timing resolution.

3. 延迟结构拆解

实时任务总延迟由多段组成：

中断延迟 + 调度延迟 + 上下文切换 + 执行时间

任何一项不可预测，都会破坏实时性边界。

4. 确定性原则

实时系统正确性取决于是否在时间窗口内完成。

超时不是性能下降，而是系统错误。

系统设计关注的是最坏情况，而非平均情况。

Timeout is a correctness failure, not performance degradation.

5. 系统级约束

时间系统直接约束三大核心：

• 调度：必须可预测
• 中断：必须可界定
• 内存：必须低抖动

因此时间系统是“系统一致性基础”。

语义总结

时间系统的本质，是将物理世界的不确定性转换为可计算约束。

系统不是运行程序，而是在维护时间正确性。

The system maintains temporal correctness rather than executing programs.
