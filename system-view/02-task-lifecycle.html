2. 任务生命周期（Task Lifecycle）

任务在 HRTOS 中的时间驱动状态流转模型

核心定义

任务生命周期描述任务从创建到终止的完整状态变化过程。

在 HRTOS 中，它不仅是状态机，而是一个受时间约束的执行实体模型。

Task lifecycle is a time-constrained execution entity model, not a simple state machine.

系统本质

在 HRTOS 中，状态的意义不是资源管理，而是时间约束管理。

每一次状态变化，本质都是一次时间语义切换：是否占用CPU、是否可调度、是否进入等待区间。

1. 创建（Created）

任务被初始化并分配TCB，但尚未进入调度体系。

关键点：是否进入系统时间模型。

2. 就绪（Ready）

任务进入就绪队列，具备执行条件但未获得CPU。

本质：等待时间分配。

3. 运行（Running）

任务获得CPU执行权，进入时间消耗阶段。

约束包括：抢占控制、WCET约束、中断延迟边界。

4. 阻塞（Blocked）

任务因资源或事件进入等待状态。

该状态本质是“时间冻结”，但仍属于系统时间模型的一部分。

5. 终止（Terminated）

任务退出系统，释放资源并移出调度体系。

关键点：是否完全脱离时间模型。

状态转移模型

Created → Ready → Running → Blocked → Ready → Running → Terminated

由三类事件驱动：中断、调度器、资源事件。

系统级意义

生命周期的核心作用是：让任务行为在时间上可预测。

若生命周期不可预测，则调度分析失效。

Lifecycle modeling enables predictable temporal behavior of tasks.

语义总结

生命周期不是状态管理，而是时间资源分配模型。

Task lifecycle is a time-resource allocation model.
