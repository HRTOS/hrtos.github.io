系统模型概述

HRTOS 是一个以时间确定性为核心约束的实时系统模型， 所有行为必须映射到可分析的时间路径。

Event → Interrupt → Scheduler → Execution → Time → Event

从时间约束系统出发的系统化学习路径（Learning Tree）

RTOS 学习总览

RTOS = 时间约束执行系统

以下结构用于建立从“理论 → 调度 → 中断 → 架构”的完整认知路径：

1. 实时系统基础 → Real-Time Model
• 硬实时 / 软实时：理解超时后果
• 时间约束：deadline / period / jitter
• WCET / WCRT：最坏情况分析模型

2. 调度系统 → Scheduling
• 抢占式调度：内核控制 CPU
• 协作式调度：任务让出 CPU

3. 中断系统 → Interrupt Model
• 外部事件如何进入内核路径

4. 内核架构 → Kernel Design
• 将机制统一回系统级结构理解
