实时问题的本质

为什么“时间”在实时系统中是第一约束

核心定义

在实时系统中，时间不是性能指标，而是系统正确性的组成部分。

如果一个任务在规定时间窗口内未完成，即使计算结果正确，也被视为错误。

时间约束的本质

实时系统的本质不是“更快”，而是“可预测”。

系统必须保证最坏情况下仍能在截止时间前完成任务。

为什么必须存在时间约束系统（核心因果）

实时系统的问题不是工程选择，而是结构性必然。

一旦系统引入外部事件（Interrupt / IO / Sensor）， 执行流就变成：

Event + Scheduling + Execution + Time Constraint

系统行为不再只依赖输入，而依赖“时间分布”。

时间与正确性的关系

正确性 = 结果 + 是否在截止时间前完成

延迟 = 逻辑错误，而不是性能问题

确定性（Determinism）

每个操作必须具备可分析的执行时间上界。

现实系统中的失败定义

• 超时未完成
• 响应超出上界
• 调度破坏时间约束

硬实时 vs 软实时

硬实时：一次超时 = 系统失败
软实时：允许偶发超时

与 System View 对应关系

核心结论

实时系统是“时间作为一等约束变量”的计算模型。
